# دليل المجلدات المطلوبة لتشغيل النظام

## الهدف
هذا المستند يوضح ما يجب نسخه أو رفعه لتشغيل نظام SawTracker بشكل صحيح، وما الذي يمكن تجاهله لأنه غير ضروري للتشغيل أو فقط خاص بالتطوير.

## المجلدات/الملفات التي يجب تحميلها

### 1) `artifacts/sawtracker/src/`
المجلد الأساسي للتطبيق.

يحتوي على:
- الصفحات الرئيسية
- المكونات UI
- المنطق التجاري
- الأدوات المساعدة
- الأنواع والواجهات
- التهيئة الخاصة بالواجهات والـ RTL

هذا المجلد ضروري بالكامل تقريباً لأن التطبيق مبني عليه.

### 2) `artifacts/sawtracker/public/` إن وجد
يضم الأصول العامة مثل:
- الصور الثابتة
- الأيقونات
- ملفات manifest
- أي ملفات يجب أن تكون متاحة مباشرة من المتصفح

### 3) ملفات الإعداد الجذرية
هذه الملفات مهمة لتشغيل المشروع:
- `package.json`
- `pnpm-lock.yaml` إذا كان موجوداً
- `tsconfig.json`
- `vite.config.ts`
- `tailwind.config.*` إن وجد
- `postcss.config.*` إن وجد
- `index.html`
- أي ملفات إعداد مرتبطة بـ Supabase أو البيئة

### 4) ملفات البيئة
- أي ملف `.env` أو قيم بيئية مطلوبة للتشغيل
- خاصة مفاتيح Supabase

### 5) مجلدات/ملفات Supabase إن وجدت
إذا كان المشروع يعتمد على SQL أو سياسات أو Edge Functions، فهذه مهمة أيضاً:
- migrations
- seed files
- policies
- functions

## المجلدات التي غالباً لا داعي لرفعها

### 1) `node_modules/`
لا يلزم تحميله، لأنه يُعاد تثبيته عبر `pnpm install`.

### 2) ملفات الكاش والنتائج المؤقتة
مثل:
- `.vite/`
- `dist/` إذا كان الهدف هو إعادة البناء من المصدر
- أي cache محلي أو build artifacts

### 3) ملفات الاختبار غير الضرورية للتشغيل
- `*.test.ts`
- `*.test.tsx`
- `*.spec.ts`
- `*.spec.tsx`

هذه مفيدة للتطوير، لكنها ليست مطلوبة لتشغيل النظام نفسه.

### 4) ملفات التوثيق غير التشغيلية
- `README.md`
- تقارير داخلية
- ملفات ملاحظات

إلا إذا كان المطلوب أرشفة المشروع بالكامل.

### 5) مجلدات التطوير الداخلية
أي مجلدات خاصة بالـ preview أو sandbox أو الأدوات المحلية غير اللازمة للإنتاج.

## ما الذي يجب اعتباره "ضرورياً" فعلاً؟
إذا كان الهدف هو تشغيل التطبيق فقط، فالأولوية تكون:
1. `src/`
2. ملفات الإعداد الجذرية
3. ملفات البيئة
4. أي أصول ثابتة مطلوبة
5. أي ملفات Supabase/DB مرتبطة بالاعتماد الحالي

## ما الذي لا يُنصح بحذفه؟
لا تحذف:
- ملفات الصفحات الأساسية
- المكونات المشتركة
- `lib/` و `utils/` إذا كانت داخل `src/`
- ملفات التهيئة
- ملفات Supabase المطلوبة للاتصال أو السياسات

## ملاحظة مهمة
إذا كان المقصود بـ "تحميلها" هو رفعها إلى مشروع جديد أو نقل المشروع إلى بيئة أخرى، فالأفضل نقل **المصدر + الإعدادات + البيئة + قاعدة البيانات المرتبطة**، وليس نسخ مجلدات التطوير أو الكاش.

## الخلاصة
### حمّل:
- `src/`
- `public/` عند الحاجة
- ملفات الإعداد
- ملفات البيئة
- ملفات Supabase/DB

### لا تحمّل:
- `node_modules/`
- الكاش
- ملفات build الجاهزة إن لم تكن مطلوبة
- ملفات الاختبار والتوثيق غير الضرورية
