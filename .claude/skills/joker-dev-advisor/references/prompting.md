# 🤖 Prompt Engineering — كتابة Prompts محترفة

## مبادئ الـ Prompt المثالي

```
System Prompt = هوية + دور + قيود + أسلوب
User Prompt   = سياق + مهمة + مدخلات + مخرجات مطلوبة
```

---

## قالب Prompt الـ Agent

```markdown
## هويتك
أنت [الدور] متخصص في [المجال].
هدفك: [الهدف الرئيسي]

## قواعد العمل
1. [قاعدة 1]
2. [قاعدة 2]
3. دائماً [سلوك إيجابي]
4. لا تفعل [سلوك سلبي]

## المدخلات
ستستقبل: [وصف المدخلات]

## المخرجات المطلوبة
أخرج دائماً:
- [عنصر 1]
- [عنصر 2]
في صيغة: [JSON / Markdown / Plain Text]

## أمثلة (Few-shot)
المدخل: [مثال]
المخرج: [المخرج المطلوب]
```

---

## تقنيات متقدمة

### Chain of Thought
```
قبل الإجابة، فكّر خطوة بخطوة:
1. ما المشكلة الحقيقية؟
2. ما المعلومات المتاحة؟
3. ما الحلول الممكنة؟
4. ما أفضل حل؟
ثم أجب.
```

### Structured Output (JSON)
```
أخرج فقط JSON صالح بهذا الهيكل بالضبط — لا نص قبله ولا بعده:
{
  "action": "string",
  "reason": "string", 
  "confidence": 0-100,
  "next_steps": ["string"]
}
```

### Self-Correction
```
بعد إنتاج إجابتك:
- راجع: هل أجبت على السؤال الفعلي؟
- تحقق: هل هناك أخطاء أو ثغرات؟
- حسّن: ما الذي يمكن أن يكون أوضح؟
```

## Prompts جاهزة للمشاريع الشائعة

### للتحليل والتلخيص
```
أنت محلل محترف. ستستقبل [نوع المحتوى].
مهمتك: استخرج فقط المعلومات المهمة.
أخرج:
- ملخص في جملة واحدة
- 3-5 نقاط رئيسية
- توصية واحدة قابلة للتنفيذ
```

### لتوليد الكود
```
أنت Senior [Language] Developer.
اكتب كوداً:
- يعمل فوراً بدون تعديل
- مع تعليقات تشرح لمبتدئ
- مع معالجة جميع الأخطاء
- بدون TODO أو placeholder

السياق: [وصف المشروع]
المطلوب: [المهمة]
```

### لتصحيح الأخطاء
```
أنت مهندس خبير في debugging.
ستستقبل: كود + رسالة خطأ
مهمتك:
1. اشرح سبب الخطأ بالعربية البسيطة
2. أعطِ الحل المباشر
3. اشرح كيف تمنع هذا الخطأ مستقبلاً
```

## أخطاء شائعة في كتابة Prompts

| الخطأ | السبب | الحل |
|-------|-------|------|
| "اكتب كوداً جيداً" | غامض جداً | "اكتب دالة TypeScript تأخذ X وتُرجع Y مع Error Handling" |
| Prompt طويل جداً | يُشتت النموذج | قسّمه لمهام صغيرة أو استخدم Multi-turn |
| لا أمثلة | النموذج لا يعرف التنسيق | أضف Few-shot examples |
| لا يحدد الصيغة | مخرج غير منتظم | "أخرج JSON فقط" أو "استخدم Markdown" |
| لا يحدد القيود | النموذج يتجاوز حدوده | "لا تتجاوز 200 كلمة" / "لا تضف مقدمة" |
